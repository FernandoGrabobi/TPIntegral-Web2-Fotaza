📸 Proyecto Fotaza 2 - Red Social de Fotografía📝 Descripción de la ConsignaFotaza 2 es una plataforma interactiva diseñada para la gestión profesional y comunitaria de contenido visual. El sistema debe permitir a los usuarios no solo compartir fotos y videos, sino también comercializar licencias, gestionar derechos de autor mediante marcas de agua y fomentar una comunidad moderada por un sistema de denuncias y validación.Objetivos Técnicos Principales:Arquitectura: Implementación de un patrón MVC (Modelo-Vista-Controlador) robusto.Renderizado: Generación de vistas dinámicas en el servidor utilizando el motor PUG.Persistencia: Base de Datos MySQL normalizada (mínimo 3FN).Seguridad: Autenticación de usuarios y protección de rutas.🏗️ Arquitectura del SistemaLa aplicación sigue una estructura de directorios modular para facilitar el mantenimiento y la escalabilidad:Shutterstockcontrollers/: Contiene la lógica de negocio y maneja las peticiones HTTP.models/: Representa las entidades de la base de datos (Usuarios, Publicaciones, Comentarios).views/: Plantillas PUG para la interfaz de usuario.routes/: Definición de los endpoints de la aplicación.middlewares/: Funciones de control de acceso y validación de datos.🛠️ Requerimientos Funcionales (MVP)1. Gestión de Usuarios y Seguridad Autenticación: Login y Registro con contraseñas encriptadas mediante bcrypt.Roles: Diferenciación entre Usuario estándar y Validador de contenidos.Seguimiento: Sistema de "Followers" para personalizar el feed de noticias.2. Publicaciones y Motor de Búsqueda CRUD Completo: Crear, leer, editar y eliminar publicaciones con múltiples imágenes y etiquetas.Filtros Avanzados: Búsqueda por autor, género, etiquetas o texto libre.Marcas de Agua: Protección automática para imágenes con copyright.3. Interacción y Moderación Valoración: Sistema de votos y visualización de promedios (estrellas).Denuncias: Si una publicación suma más de 3 denuncias, pasa a revisión del Validador.Notificaciones: Alertas al usuario por nuevos comentarios, seguidores o intereses de compra.📊 Modelo de Datos (Diagrama ER)El diseño de la base de datos es relacional y garantiza la integridad referencial:Users: Perfiles, biografía y estadísticas de seguidores.Posts: Contenido multimedia, descripciones y tipos de licencia.Comments: Relación muchos-a-uno con publicaciones.Denuncias/Reports: Registro de motivos y estado de la revisión.🚀 Instalación y ConfiguraciónRequisitos PreviosNode.js instalado.Servidor MySQL activo.PasosClonar: git clone https://github.com/usuario/fotaza2.gitDependencias: npm install.Ambiente: Crear archivo .env basado en .env.example.Base de Datos: Ejecutar npm run db:init para crear tablas y cargar semillas (seeds).Ejecutar: npm start (o npm run dev para usar nodemon).👨‍💻 Usuarios de Prueba para Evaluación Admin/Validador: admin@fotaza.com / admin123Usuario 1: test1@fotaza.com / user123Usuario 2: test2@fotaza.com / user123
